# Aurora-Executor

Aurora-Executor enables users to quickly provision a Kali machine into an attack workstation suitable for attack simulation, install the necessary tools along with our Python interface, and thereby rapidly and semi-automatically execute attack chains generated by Aurora.

## 🚀 Quick Start

### One-Click Deployment (Recommended)

Run this command directly on your Kali/Ubuntu machine:

```bash
# Download and execute the deployment script
wget -qO- https://raw.githubusercontent.com/LexusWang/Aurora-executor/main/auto_deploy.sh | bash
```

Or download first, then execute:

```bash
# Download the script
wget https://raw.githubusercontent.com/LexusWang/Aurora-executor/main/auto_deploy.sh

# Make it executable
chmod +x auto_deploy.sh

# Run the script
bash auto_deploy.sh
```

### What the Script Does

The `auto_deploy.sh` script automatically:
1. ✅ Installs system dependencies (virtualenv, expect, tmux)
2. ✅ Creates a Python virtual environment
3. ✅ Installs attack-executor library and dependencies from PyPI
4. ✅ Downloads Sliver C2 server (sliver-server_linux-amd64)
5. ✅ Configures Sliver multiplayer server
6. ✅ Starts Metasploit RPC service
7. ✅ Starts Sliver C2 server in tmux sessions

## 📋 System Requirements

- **OS**: Kali Linux, Ubuntu 20.04+, Debian 11+
- **Python**: 3.11 or higher
- **Privileges**: sudo access required
- **Network**: Internet connection for downloading dependencies

## 🛠️ Installation Methods

### Method 1: Clone Repository (For Development)

```bash
# Clone the repository
git clone https://github.com/LexusWang/Aurora-executor.git

# Navigate to the directory
cd Aurora-executor

# Run the deployment script
bash auto_deploy.sh
```

### Method 2: Direct Download and Execute

```bash
# Using wget
wget https://raw.githubusercontent.com/LexusWang/Aurora-executor/main/auto_deploy.sh
bash auto_deploy.sh

# Using curl
curl -O https://raw.githubusercontent.com/LexusWang/Aurora-executor/main/auto_deploy.sh
bash auto_deploy.sh
```

### Method 3: Manual Installation

```bash
# 1. Install system dependencies
sudo apt-get update
sudo apt-get install -y virtualenv expect tmux

# 2. Create virtual environment
virtualenv env_aurora-executor

# 3. Install Python packages
env_aurora-executor/bin/pip install attack-executor==0.2.7 questionary==2.1.0

# 4. Download Sliver server
mkdir -p attack_tools
cd attack_tools
wget https://github.com/BishopFox/sliver/releases/download/v1.7.2/sliver-server_linux-amd64
chmod +x sliver-server_linux-amd64
cd ..

# 5. Configure and start services
# Follow the configuration steps in auto_deploy.sh
```

## 📦 What Gets Installed

### Python Packages
- **attack-executor** (0.2.7): Core library for attack automation
- **questionary** (2.1.0): Interactive CLI prompts
- **sliver-py** (>=0.0.19): Sliver C2 Python client
- **pymetasploit3** (>=1.0.6): Metasploit RPC client
- **rich** (>=14.0.0): Terminal formatting

### Tools
- **Sliver C2 Server**: Command and control framework
- **Metasploit Framework**: Penetration testing platform (must be pre-installed)

## 🎮 Usage

After successful deployment:

```bash
# Activate the virtual environment
source env_aurora-executor/bin/activate

# View Metasploit session
tmux attach -t msf

# View Sliver session
tmux attach -t sliver

# Detach from tmux: Ctrl+B, then D
```

### Running Attack Chains

```bash
# Execute an attack chain
python attack.py

# Or use specific attack chain scripts
python attack_chains/SAM_password_hash_cred_known-1/execution_SAM_password_hash_cred_known-1.py
```

## 🔧 Configuration

The deployment script creates a `config.ini` file with default settings:

```ini
[sliver]
client_config_file = /path/to/zer0cool.cfg

[metasploit]
password = 123456
host_ip = 127.0.0.1
listening_port = 55552
```

Modify these settings as needed for your environment.

## 🐛 Troubleshooting

### Metasploit RPC Not Starting
```bash
# Check if msfconsole is installed
which msfconsole

# Manually start Metasploit RPC
msfconsole -q -x "load msgrpc Pass=123456"
```

### Sliver Server Issues
```bash
# Check if Sliver is running
tmux list-sessions

# View Sliver logs
tmux attach -t sliver
```

### Permission Denied Errors
```bash
# Ensure script has execute permissions
chmod +x auto_deploy.sh

# Run with sudo if needed
sudo bash auto_deploy.sh
```
